  • Accessibility Levels in Java:
    • Private:
      • Accessible only within the class where it is defined.
      • Cannot be accessed from outside the class, not even by subclasses or other classes in the same package.
    • Default (Package-Private):
      • If no access modifier is specified, the member is accessible only within its own package.
      • Accessible by any class within the same package but not from classes in different packages.
    • Protected:
      • Accessible within the same package and by subclasses in other packages.
      • Allows subclasses to access members of the superclass, even if they are in different packages.
    • Public:
      • Accessible from any other class, regardless of the package.
      • No restrictions on access, making it the most permissive access level.
  • Tricky Interview Questions:
    • Question: "Can a subclass in a different package access a protected member of its superclass?"
    • Answer: "Yes, a subclass in a different package can access a protected member of its superclass, but it cannot access it through an instance of the superclassâ€”only through inheritance."
    • Explanation: "Protected members are accessible within the same package and by subclasses, even if the subclass is in a different package. However, outside the package, access must be through inheritance (i.e., `super.memberName`)."
    • Question: "What happens if you don't specify an access modifier in Java?"
    • Answer: "If no access modifier is specified, the member has default (package-private) access. This means it is accessible only within the same package."
    • Explanation: "The default access level is often overlooked. It's important to remember that without an explicit modifier, access is restricted to the package level."
    • Question: "Can a private method in a superclass be overridden by a subclass?"
    • Answer: "No, a private method in a superclass cannot be overridden by a subclass because private methods are not inherited by subclasses."
    • Explanation: "Since private methods are not visible to subclasses, they cannot be overridden. The subclass can define a new method with the same name, but it's not considered an override."
    • Question: "Is it possible to reduce the visibility of an overridden method in a subclass?"
    • Answer: "No, you cannot reduce the visibility of an overridden method. The overridden method in the subclass must have the same or greater visibility."
    • Explanation: "For example, if a method in the superclass is public, the overriding method in the subclass must also be public. You cannot make it protected or private."
