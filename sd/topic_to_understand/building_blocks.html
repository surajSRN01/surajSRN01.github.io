System design building blocks using cloud service

  1. Domain Name System (DNS):

    DNS is a system that translates human-readable domain names into IP addresses, making it essential for locating and accessing websites on the internet.
    Tools like AWS Route 53 or Google Cloud DNS are commonly used for implementing DNS in cloud services.

  2. Load Balancers:

    A load balancer distributes incoming network traffic across multiple servers, ensuring optimal resource use, reducing server overload, and improving system reliability by bypassing failed servers.
    In cloud environments, services like AWS Elastic Load Balancing (ELB) and Google Cloud Load Balancing are often utilized.

  3. Databases:

    This block involves storing, retrieving, modifying, and deleting data as part of various data-processing tasks. The discussion includes database types, replication, partitioning, and analysis of distributed databases.
    Popular cloud databases include Amazon RDS, Google Cloud SQL, and Azure Cosmos DB.

  4. Key-Value Store:

    A non-relational database that stores data in key-value pairs. This section explains the design of a key-value store, focusing on scalability, durability, and configurability.
    Common tools in cloud services include Amazon DynamoDB and Redis.

  5. Content Delivery Network (CDN):

    A CDN efficiently delivers content such as videos, images, audio, and web pages to end-users while minimizing latency and reducing the burden on data centers.
    Cloud-based CDNs like Amazon CloudFront and Azure CDN are widely used.

  6. Sequencer:

    This block focuses on designing a system for generating unique IDs, with an emphasis on maintaining causality.
    Techniques such as using UUIDs or Snowflake ID generators (used by Twitter) are explored.

  7. Service Monitoring:

    Monitoring systems are critical in distributed systems for analyzing performance and alerting stakeholders in case of issues.
    Here, both server-side and client-side monitoring systems are built using tools like Amazon CloudWatch, Prometheus, and Grafana.

  8. Distributed Caching:

    In this block, we design a distributed caching system where multiple cache servers coordinate to store frequently accessed data.
    Tools like Amazon ElastiCache (for Redis or Memcached) are used for implementing distributed caching in the cloud.

  9. Distributed Messaging Queue:

    This section covers the design of a messaging queue consisting of multiple servers, facilitating communication between producers and consumers. It helps with independent scalability and reliability.
    Cloud services like Amazon SQS and Apache Kafka are commonly used.

  10. Publish-Subscribe System:

    The publish-subscribe (pub-sub) system is an asynchronous communication method popular in serverless and microservices architectures.
    Tools like Google Cloud Pub/Sub and Amazon SNS are commonly used in the cloud for implementing pub-sub systems.

  11. Rate Limiter:

    This block discusses the design of a system that throttles incoming requests based on predefined limits, protecting services from excessive usage.
    Rate limiting can be implemented using API Gateway services like Amazon API Gateway with built-in rate-limiting features.

  12. Blob Store:

    A storage solution for unstructured data, such as multimedia files and binary executables.
    Cloud-based blob storage solutions include Amazon S3 and Azure Blob Storage.

  13. Distributed Search:

    This block covers the design of a search system that processes user queries and returns relevant content quickly. The discussion includes components like crawling, indexing, and searching.
    Tools like Elasticsearch and Amazon OpenSearch are commonly used.

  14. Distributed Logging:

    Designing a system for efficient logging in a distributed environment is crucial due to the I/O-intensive nature of logging operations.
    Cloud-based logging tools like Amazon CloudWatch Logs, Google Cloud Logging and Elasticsearch are often utilized for this purpose.

  15. Distributed Tracing:

    Distributed tracing helps in tracking the flow of requests across different services in a distributed system, allowing developers and operators to pinpoint performance bottlenecks and troubleshoot issues.
    Cloud-based tracing tools like AWS X-Ray, Google Cloud Trace and Zipkin are often utilized for this purpose.

  16. Distributed Task Scheduling:

    This block focuses on designing a distributed task scheduler that allocates resources to tasks, ensuring that task-level and system-level goals are met.
    Cloud services like Amazon Step Functions and Apache Airflow are commonly used for task scheduling.

  17. Sharded Counters:

    Sharded counters are used for efficiently counting millions of concurrent read/write requests, such as likes on social media posts. Techniques for implementing sharded counters include using distributed databases or specialized tools like Google Cloud Spanner.
